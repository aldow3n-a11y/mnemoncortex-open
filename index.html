Architecture

Conversation → Capture → Graph

Every message flows through a saliency pipeline that extracts insights, structures them, and writes them to a persistent graph — not just a flat text file.

01

Observe

Regex capture scans for 21 saliency patterns — "lesson learned:", "root cause was", "decided to", etc.

→
02

Extract

LLM extraction pulls structured (subject, relation, object) triples with confidence scores.

→
03

Distill

Queue flushes → writes to daily log + Mnemon graph with category, importance, and entities.

04

Link

Auto-edges: temporal (same batch), entity (shared entities), semantic (cos ≥ 0.80). Causal candidates returned for LLM review.

→
05

Recall

Intent-aware beam search: WHY → causal chains, WHEN → temporal, ENTITY → co-occurrence, GENERAL → RRF fusion.

→
06

Decay

Effective importance decays over time. Access counts boost retention. GC prunes below threshold.

Features

Not just a database. A memory system.

Built for agents that need to remember why things happened, not just that they happened.

🧠

Intent-Aware Recall

Ask "why did X break?" and Mnemon traverses causal chains, not keyword matches. WHY, WHEN, ENTITY, GENERAL — each intent triggers a different graph traversal strategy.

WHYWHENENTITYGENERAL
🔗

4-Graph Edges

Temporal (backbone + proximity), Entity (shared entities), Causal (reason → effect), Semantic (cosine ≥ 0.80). Each edge type serves a distinct recall path.

temporalentitycausalsemantic
⚡

Auto-Capture Pipeline

21 regex patterns + LLM triple extraction running on every message. Zero-config insight capture — your agent remembers without being told to remember.

🔄

Auto-Dedup & Link

Writing the same insight twice? Skipped. Similar but better? Auto-replaced. Same-batch items get temporal edges. Shared entities get entity edges. No manual curation needed.

📉

Importance Decay

Every insight has an importance score that naturally decays. Frequently recalled insights stay fresh. GC prunes what the agent has outgrown. Your memory stays lean.

🏠

Local-First, No Cloud

SQLite WAL database. Ollama embeddings on your machine. No API calls for recall. Your agent's memory stays private and fast, with zero cloud dependency.

Quickstart

Up and running in 3 commands

bash
# Install Mnemon brew install mnemon-dev/tap/mnemon # or: go install github.com/mnemon-dev/mnemon@latest # Set up agent integration mnemon setup --target openclaw --yes # Pull embedding model ollama pull nomic-embed-text
bash
# Store a decision mnemon remember "Switched to pm2 for process mgmt" \ --cat decision --imp 4 \ --entities "pm2,systemd,deployment" # Recall why something broke mnemon recall "why did the deploy fail" \ --intent WHY --limit 3 # Link a cause mnemon link <cause-id> <effect-id> \ --type causal --weight 0.8
Integrations

Works with any agent framework

Native integration via hooks, plugins, and skills. Bootstrap injection, remind/nudge before prompt build, compact before context compression. One-command setup: mnemon setup --target openclaw --yes
